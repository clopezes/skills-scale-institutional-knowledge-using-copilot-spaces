# OctoAcme — Cross-role Handoff Checklist

Use this checklist to make handoffs explicit between planning, implementation, release, and support.
Pair it with [Roles and Personas](octoacme-roles-and-personas.md), [Execution and Tracking](octoacme-execution-and-tracking.md), and [Release and Deployment](octoacme-release-and-deployment.md).

## 1) Product + Planning handoff (Product Manager → Delivery Team)
- [ ] Problem statement and desired outcomes documented
- [ ] Acceptance criteria and out-of-scope items confirmed
- [ ] Priority, timeline, and dependencies aligned with Project Manager
- [ ] UX deliverables (or timeline for delivery) clarified
- [ ] Security/compliance constraints called out
- [ ] Metrics and instrumentation expectations defined

## 2) Design handoff (UX/UI Designer → Engineering)
- [ ] Final designs/wireframes linked in project artifact
- [ ] Accessibility requirements captured
- [ ] Edge states and error states included
- [ ] Open UX decisions listed with owner and due date

## 3) Build handoff (Engineering Manager + Developers → QA/Release)
- [ ] Implementation complete against acceptance criteria
- [ ] Test evidence documented (automated + manual where needed)
- [ ] Known limitations or risks documented
- [ ] Rollback considerations reviewed with DevOps / SRE
- [ ] Security checks completed for sensitive changes

## 4) Release handoff (Project Manager + DevOps / SRE → Stakeholders/Support)
- [ ] Go/no-go decision documented with responsible approvers
- [ ] Release notes and customer-facing summary prepared
- [ ] Support playbook/FAQ updates completed
- [ ] Monitoring and alert ownership confirmed
- [ ] Incident escalation path confirmed for release window

## 5) Post-release handoff (Support + Analytics → Product/Project)
- [ ] Initial support signals reviewed (first 24–72h)
- [ ] Adoption and reliability metrics reviewed
- [ ] Follow-up fixes/improvements captured in backlog
- [ ] Retrospective action items assigned with owners and dates
