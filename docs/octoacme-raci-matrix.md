# OctoAcme — Lightweight RACI Matrix

Use this matrix with [Roles and Personas](octoacme-roles-and-personas.md), [Project Planning](octoacme-project-planning.md), and [Release and Deployment](octoacme-release-and-deployment.md) to reduce ownership ambiguity during delivery.

## Common lifecycle activities

| Activity | Product Manager | Project Manager | Engineering Manager / Tech Lead | Developers | UX/UI Designer | Security / Compliance Lead | DevOps / SRE | Customer Support / Success Representative | Data Analyst / Analytics Partner |
|---|---|---|---|---|---|---|---|---|---|
| Define problem statement and outcomes | A | C | C | I | C | I | I | C | C |
| Prioritize backlog and release scope | A | R | C | C | C | C | C | C | C |
| Finalize implementation plan and estimates | C | A | R | R | C | C | C | I | I |
| Validate UX acceptance and accessibility | C | I | C | C | A/R | I | I | C | I |
| Security and compliance readiness review | I | C | C | R | I | A | C | I | I |
| Release readiness and go/no-go | C | A | R | C | I | C | R | C | C |
| Publish release notes and stakeholder communication | C | A | I | I | I | I | C | R | I |
| Post-release monitoring and incident triage | I | C | C | R | I | C | A/R | C | C |
| Review outcomes and retrospective actions | A | R | C | C | C | I | C | C | R |

## Legend
- **R** = Responsible (does the work)
- **A** = Accountable (owns final decision/outcome)
- **C** = Consulted (provides input)
- **I** = Informed (kept updated)
